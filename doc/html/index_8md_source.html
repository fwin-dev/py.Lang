4 # Summary of functionality
5 
6 This package provides a lot of miscellaneous things that probably should have been included in python's built-in
7 library but weren't, including:
8 
9 * Various function/method utilities/tools
10  * An abstract class method
11  * Timing execution of a function with a decorator
12  * A way to get the argument names and values of a function without using **kwargs or *args
13  * A way to describe the argument and return types of functions, kind of similar to a statically typed language
14 * Various class utilities/tools
15  * Getting the variables of a class instance with __slots__
16 * Various module and package utilities/tools (Is a module built in? In what file is it? etc.)
17 * Implementation of various structures to hold data
18  * LIFO/Stack
19  * Frozen dictionary
20  * Ordered set
21 * A peekable iterator
22 * An improved differ, based on `difflib.SequenceMatcher`
23 * An improved ArgParser for parsing command line arguments
24 * A poor man's debug tracer when a better tracer isn't available (for example, when running a python script over ssh)
25 * Easy event handling with subscription/listening, including event logging
26 * Terminal/user interaction improvements

32 ### An abstract class method
33 
34 The built in `abc.abstractmethod` decorator won't work if used in conjunction with a `classmethod` decorator.
35 Use this abstract method decorator instead:
36 
37  from Lang.FuncTools.Abstraction import abstractmethod
38 
39  @classmethod
40  @abstractmethod
41  def myFunction(...):
42  asdf
43 
44 Make sure that `abstractmethod` comes after `classmethod`, else you will get:
45 `AttributeError: 'classmethod' object has no attribute '__name__'`
46 
47 ### Timing execution of a function
48 
49 `timeIt` is a function decorator, so with the function you want to time, do:
50 
51  from Lang.FuncTools import timeIt
52 
53  @timeIt
54  def myFunction(...):

57 ### Getting a function's arguments
58 
59 Python provides `**kwargs` and `*args` to get a dictionary or list of a function's arguments, but this makes it hard
60 for IDEs and documentation generators to determine all possible arguments to the function. As an alternative to
61 `**kwargs` or `*args`, you can use the `getArgs()` method:
62 
63  from Lang.FuncTools import getArgs
64  def myFunc(arg1, arg2):
65  allArgs = getArgs()
66 
67 This method uses python's built in `inspect` module to go up the stack and inspect arguments.
68 
69 In the case above, `allArgs` will be a list of values, similar to as if `*args` was used. To get a dictionary of
70 argument names and values instead, similar to `**kwargs`, use `getArgs(useKwargFormat=True)`.
71 
72 Note that `cls` and `self` are automatically ignored for class methods and instance methods.
73

172 ### Ordered set
173 
174 The built-in python `set` is just like a `list`, except for 2 things:
175 
176 - Sets can't contain duplicate elements
177 - Sets are unordered
178 
179 However, there are some cases where an ordered set (aka a list with no duplicate elements) is desirable. For this, use
180 the `OrderedSet` provided here, which provides a similar implementation compared to the built-in `set`, but also provides
181 methods typically found in a list, such as `insert`/`insertAt`. Refer to the source and unit test for a list of all methods.
182 
183  from Lang.Struct import OrderedSet
184  set_ = OrderedSet(range(1,10))

186 ## Peekable iterator
187 
188 Want to use an iterator for RAM usage concerns, but need to know the next element sometimes without advancing the
189 iterator? Then this "peekable iterator" implementation is for you! Just wrap any iterable inside a `PeekableIterable`.
190 That inner iterable is most likely a pure iterator itself (such as a generator), but it could also be anything that
191 implements `__iter__`.
192 
193  from Lang.Iter import PeekableIterable
194  def asdf():
195  for i in range(0,10):
196  yield i
197  nums = PeekableIterable(asdf())
198  print(nums.next()) # advances the iterator
199  print(nums.peek()) # peeks without advancing
200  print(nums.hasNext()) # checks if there's a next element

202 ## Improved differ
203 
204 This differ builds upon `difflib.SequenceMatcher` which can diff any python objects with `__eq__` implemented, contained
205 within a list, tuple, etc. However, the built in class only provides methods for getting matching sets of indicies which
206 refer to matching elements, leaving you to infer which indicies don't match. It also doens't give you direct access to
207 elements that do or don't match. This differ adds all of that functionality. It also fixes:
208 
209 - A bug in the built in differ where subsequent calls to `get_matching_blocks()` returns results in a different format
210 - Calculating the similarity ratio: The built in differ calculates this ratio taking both diff sides into account, but what's usually wanted is how much one side is similar/different compared to the other side, i.e. `(1 - ratio) / 2 + ratio`
211 
212  from Lang.Diff import SequenceMatcher
213  diff = SequenceMatcher(tuple("aebcdef"), tuple("abbcdgef"))
214  print(list(diff.get_matching_blocks()))
215  print(list(diff.get_mismatching_blocks()))
216  print(list(diff.get_matching_elems()))
217  print(list(diff.get_mismatching_elems()))
218 
219  Prints the following:
220  [BlockMatch(a={index=0,size=1}, b={index=0,size=1}), BlockMatch(a={index=2,size=3}, b={index=2,size=3}), BlockMatch(a={index=5,size=2}, b={index=6,size=2})]
221  [BlockMismatch(a={index=1,size=1}, b={index=1,size=1}), BlockMismatch(a=None, b={index=5,size=1})]
222  [ElemMatch(a=('a',), b=('a',)), ElemMatch(a=('b', 'c', 'd'), b=('b', 'c', 'd')), ElemMatch(a=('e', 'f'), b=('e', 'f'))]
223  [ElemMismatch(a=('e',), b=('b',)), ElemMismatch(a=None, b=('g',))]
224 
225 More functions are available, including:
226 
227 - `getmatching()` and `getmismatching()`
228  - These return structures with `.block` and `.elems` attributes containing both block indicies and the elems which the block refers to
229 - `get_matching_elems_useOnce()` and `get_mismatching_elems_useOnce()`
230  - These are the same as `get_matching_elems()` and `get_mismatching_elems()` except that they are generators instead of functions returning a list

232 ## An improved ArgParser
233 
234 In addition to the plethora of features in python's built in `ArgParser`, a few more are added in here:
235 
236 - Improved help formatting, similar to `man`
237 - 3 way boolean (`True`, `False`, `None`)
238 - Required named parameters - the built in ArgParser only supports required positional arguments and optional named parameters
239 
240 The new `ArgParser` uses the same interface as the old one, so see the built in `ArgParser` documentation.
241 
242 Example:
243 
244  from Lang.ArgParser import ArgParser
245  parser = ArgParser(argument_default=None, add_help=True, description="Adds a user to a linux machine")
246  parser.add_argument("username")
247  parser.add_argument("-p", "--password", required=False, help="Prompt for password if this is not given")
248  parser.add_argument("-H", "--create-home", type=Bool3Way, required=True,
249  help="Controls home directory creation for user. None uses the default behavior which varies between machines.")
250  args = parser.parse_args()

259 ## Event handling
260 
261 For general event handling with event subscription and listeners, there is a proxy API. The proxy receives an event
262 (via a method call) and then calls any subscribers (aka receivers) to the event. In this implementation, a subscriber
263 subscribes to all events, but only chooses to implement the methods for the events that it is interested in. The proxy
264 checks each receiver to see if it has implemented the method, and if so, calls the method. All receivers can be
265 forced to implement all methods by setting `errorOnMethodNotFound=True`.
266 
267  from Lang.Events.Proxy import EventProxy, EventReceiver
268  proxy = EventProxy(errorOnMethodNotFound=False)
269 
270  class Foo(EventReceiver):
271  def someEvent(self, parametersOfEvent, moreParams=None):
272  print("I was called")
273 
274  foo = Foo()
275  proxy.addReceiver(foo)
276  proxy.someEvent("abc", moreParams=123)

278 ## Handling uncaught exceptions
279 
280 When an uncaught exception happens, a special `notifyException` method will be called on each EventReceiver
281 automatically, if the method is implemented, with the exception instance and a traceback instance as parameters.
282 What you do with these parameters is up to you, but here is an example:
283 
284  class Foo(EventReceiver):
285  def notifyException(self, exceptionInstance, tracebackInstance):
286  import traceback
287  tracebackStr = "".join(traceback.format_tb(tracebackInstance))
288  exceptionStr = str(exceptionInstance)
289  print(tracebackStr)
290  print()
291  print(exceptionStr)
292 
293 This idea can be combined with event logging, shown below.
294 Also note that `errorOnMethodNotFound` has no effect on `notifyException`, as it is a special case.
295 
296 ### Event logging
297 
298 Python has decent built in logging, but it doesn't follow standard object-oriented concepts where methods represent
299 actions, so the API is not ideal for recording different events in a heavily event based system, as there would need
300 to be a special, separate call to the logging API for every event. The logging API in `Lang.Events.Logging` fixes this.
301 It uses the event handling API shown above, where the `Logging` class is an `EventProxy`, and the loggers are
302 `EventReceiver`s.
303 
304 #### Example using StdoutLogger
305 
306 This is a very simplistic example of logging to stdout:
307 
308  from Lang.Events.Logging import Logging, StdoutLogger
309  log = Logging(StdoutLogger)
310  log.notifyMyEvent("details", "in", "arguments", "here")
311 
312 #### Example using a custom logger and/or multiple loggers
313 
314 When using multiple loggers, the function you call on the `Logging` instance will be called on every logger.
315 
316  from Lang.Events.Logging import Logging, LoggerAbstract, StdoutLogger
317  class MyFileLog(LoggerAbstract):
318  def __init__(self, filePath):
319  super(MyLogger, self).__init__()
320  self._filePath = filePath
321  def notifyFolderCheck(self, folder):
322  with open(self._filePath, "a") as file_:
323  file_.write("Checking folder: " + folder)
324 
325  log = Logging((StdoutLogger, MyFileLog))
326  log.notifyFolderCheck("folder/path/here")

338 ### Using formatted text on the terminal
339 
340  from Lang.Terminal import FormattedText
341 
342 `FormattedText` returns a subclass of the built in python `str` type. It adds terminal formatting codes when
343 the `str` function is called on it:
344 
345  boldStr = str(FormattedText("I'm in bold", bold=True))
346 
347 `boldStr` will contain some funny looking characters which enable formatting, but these characters do not influence
348 any other function of the string. For example:
349 
350  char = boldStr[2]
351 
352 will store `m` into `char`. Similarly, `len` and other functions will act the same as if the string were a direct
353 instantiation of `str`.
354 
355 Many more text attributes are available, including coloring - see the source for more information.

357 ### Tables
358 
359 Printing tabular data to the terminal is very common. This implementation of a table has a couple extras with it.
360 Here is an example:
361 
362  from Terminal import Table
363  table = Table()
364  table.setColHeaders(("First Name", "Last Name Initial", "Number")) # optional
365  table.setColMaxLens([None] * 3) # enables automatic column sizing for the 3 columns
366  table.addRow(["Jesse", "C", str(1234)]) # explicitly convert all elements to strings
367  table.printLive() # prints all rows (and headers) that haven't been printed before
368 
369 If another row is added to the table, another call to `printLive()` will only print only that new row.
